# 클로바노트 회의록 요약

클로바노트 음성 기록 전문(.txt)을 읽고 구조화된 회의록 요약을 마크다운으로 생성한다.

## 워크플로우

1. 사용자가 클로바노트 .txt 파일 경로를 제공하면 Read 도구로 파일을 읽는다
2. 전문을 파싱하여 메타데이터(제목, 날짜, 참석자)와 발언 내용을 추출한다
3. 아래 출력 형식에 맞춰 회의록 요약을 생성한다

파싱 시 클로바노트 형식이 불명확하면 [clovanote-format.md](references/clovanote-format.md) 참조.

## 출력 형식

```markdown
# 회의록: {회의 제목}

## 기본 정보
- **일시**: {날짜 및 시간}
- **소요 시간**: {녹음 시간}
- **참석자**: {참석자 목록}

## 회의 개요
{전체 회의의 맥락과 목적을 2~3문장으로 요약}

## 주요 논의사항
### 1. {주제명}
{주제에 대한 맥락과 배경 1~2문장}

> **{발언자}**: "{핵심 발언 인용 또는 요약}"

> **{발언자}**: "{이에 대한 반응/의견}"

{논의 흐름 요약. 어떤 맥락에서 어떤 결론으로 이어졌는지}

**결정**: {이 주제에서 합의된 사항}
**액션**: {담당자} - {할 일} ({기한/맥락})

### 2. {주제명}
...

## 종합 결정사항
- {주제별 결정사항 중 특히 중요한 것 재정리}

## 액션 아이템 종합
| 담당자 | 할 일 | 비고 |
|--------|--------|------|
| {이름} | {구체적 업무} | {기한/맥락} |

## 미결 이슈
- {논의되었으나 결론나지 않은 사항}

## 후속 일정
- {다음 미팅, 워크숍 등 예정된 일정}
```

## 요약 원칙

- 음성 인식 오류를 자연스러운 문장으로 보정한다 (예: "tl dl" → "TL;DR", "아이씨" → "IC")
- 인명의 음성 인식 오류는 문맥으로 판단하여 일관되게 표기한다 (예: 같은 사람이 "재도", "제도"로 혼용되면 하나로 통일)
- "참석자 N" 형태의 미식별 화자는 그대로 유지한다
- 민감한 내용(퇴사 의향, 개인 감정 등)도 사실 그대로 기록한다. 회의록은 내부 참고용이다
- 발언의 의도와 맥락을 살려 요약한다. 단순 나열이 아닌 논의의 흐름을 보여준다
- 각 주제 안에서 발언 인용(blockquote)과 서술 요약을 교차 배치한다. 인용은 논의의 온도와 뉘앙스를 전달하고, 서술은 흐름과 맥락을 정리한다
- 한 주제에 발언 인용은 2~4개가 적절하다. 모든 발언을 인용하지 않는다
- 비즈니스 용어, 직함, 프로젝트명은 원문 그대로 유지한다
